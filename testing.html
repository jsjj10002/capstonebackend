  • 간소화된 입력: 일기 내용만으로 모든 정보 추출

✍️ 일기 작성 - 간소화된 방식 (v3.0.1)

POST /api/diaries - 간소화된 워크플로우

v3.0.1 간소화된 워크플로우

프론트엔드 요청 방식: selectedPersonId 없이도 이미지 생성 가능

  1. 일기 내용 입력 (@태그 지원)
  2. 장면 묘사 자동 생성
  3. 외모 키워드 + 성별 입력 (인물 선택 없이)
  4. 동기적 이미지 생성 (완료 후 응답)

특징: 기존 인물 DB 없이도 바로 이미지 생성 가능

v3.0.1 간소화된 워크플로우

  • 기존 인물 DB 연동 없이 바로 사용 가능
  • 프론트엔드 요청 데이터만으로 이미지 생성
  • 동기적 처리로 완성된 이미지와 함께 응답
  • selectedPersonId 필드 불필요

✍️ 일기 작성 - 기존 방식 (v3.0)

POST /api/diaries

v3.0 정확한 6단계 워크플로우

  1. 일기 작성 (@태그 지원)
  2. 장면 묘사 생성 (JSON 응답, @태그 기반)
  3. 주요 인물 선택 (기존 인물 목록에서 사용자가 직접 선택)
  4. 주요 인물 외형 작성 (선택된 인물의 성별 정보 자동 사용)
  5. 프롬프트 생성 (장면 묘사 100% 활용)
  6. 이미지 생성

v3.0 통합 워크플로우

단계: 1) 일기 작성 → 2) 장면 묘사 생성 → 3) 주요 인물 선택 → 4) 외모 키워드 입력 → 5) 이미지 생성

v3.0 워크플로우 안내

  1. 일기 내용에 @태그 포함하여 작성
  2. "장면 묘사 자동 생성" 버튼 클릭
  3. 기존 인물 목록에서 주요 인물 선택
  4. 외모 키워드 입력 후 일기 작성 완료
